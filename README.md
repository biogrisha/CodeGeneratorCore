# Code Generation Pipeline

## Этапы генерации кода

Процесс генерации кода состоит из следующих этапов:

1. **Парсинг XML в промежуточное представление**
   На этом этапе входной XML-документ разбирается и преобразуется в удобную для дальнейшей обработки внутреннюю структуру данных.

2. **Формирование e-графа программы из термов**
   Построение e-графа на основе полученных термов. Это позволяет эффективно представлять эквивалентные выражения и выполнять преобразования.

3. **Применение e-matching**
   Выполняется e-matching по алгоритму, описанному в статье:
   https://leodemoura.github.io/files/ematching.pdf
   Используется собственная реализация, разработанная в рамках проекта системы компьютерной алгебры.

4. **Сатурация и упрощение выражений**
   Применяются правила переписывания с целью насыщения (saturation) e-графа и упрощения выражений, по аналогии с подходом, используемым в egg.

5. **Составление конечного документа**
   На основе оптимизированного представления формируется итоговый документ или сгенерированный код.

## Примеры

Примеры генерации файлов находятся в папке `TestFiles`.

Для каждого XML-файла предусмотрены соответствующие C++-файлы:

* версия **с оптимизацией**
* версия **без оптимизации**
